🚀 GitHub Pages 一键部署指南

1 注册GitHub账号

如果你还没有GitHub账号，需要先注册：

// 访问GitHub官网注册
https://github.com/signup

注册要点：

  • 用户名建议用英文，如：zhangsan123
  • 记住用户名和密码
  • 验证邮箱地址

2 创建新仓库

登录GitHub后，创建新仓库：

仓库设置说明

仓库名称：建议用英文，如 `relationship-survey`

描述：可填写"关系健康度问卷系统"

公开/私有：选择Public（公开），这样GitHub Pages才能访问

初始化选项：不勾选 "Add a README file"

// 仓库创建后地址示例
https://github.com/[你的用户名]/[仓库名]

3 上传文件（三种方法）

方法A：拖拽上传（最简单）

在你的仓库页面：

  1. 点击"Add file" → "Upload files"
  2. 将`survey`文件夹下的所有文件拖拽到上传区域
  3. 填写提交信息，如："添加问卷系统文件"
  4. 点击"Commit changes"

方法B：使用Git命令

# 安装Git，然后在命令行执行：
git clone https://github.com/[用户名]/[仓库名].git
cd [仓库名]
# 复制所有文件到仓库
cp -r [问卷文件夹路径]/* .
git add .
git commit -m "添加问卷系统"
git push origin main

方法C：使用GitHub Desktop

下载GitHub Desktop客户端，图形化操作更简单。

4 开启GitHub Pages

详细步骤：

  1. 进入你的仓库页面
  2. 点击上方"Settings"（设置）
  3. 在左侧菜单找到"Pages"（页面）
  4. 在"Source"（来源）中选择："Deploy from a branch"（从分支部署）
  5. 在"Branch"（分支）中选择：main 分支
  6. 选择文件夹：/ (root)
  7. 点击"Save"（保存）

等待部署：GitHub会自动构建，大约1-2分钟后会显示绿色部署成功标志。

部署成功后访问地址：

https://[你的用户名].github.io/[仓库名]/index.html

重要链接：

  • 📋 问卷页面：https://.../index.html
  • ⚙️ 管理后台：https://.../admin.html
  • 🚀 配置应用：https://.../apply_config.html

💡 部署完成后的测试

  1. 用不同设备（手机、电脑）访问问卷地址
  2. 测试问卷填写和提交功能
  3. 测试管理员后台登录和查看数据
  4. 测试移动数据网络访问（应该可以了）

⚠️ 重要注意事项

1. 数据存储限制：GitHub Pages是静态托管，数据仍然保存在浏览器localStorage中，不同设备数据不互通。

2. 安全性：管理员密码在代码中明文存储，如需更高安全性需要后端支持。

3. 更新问卷：修改问卷后需要重新上传文件到GitHub并等待部署。

✅ 部署检查清单

  • ✅ GitHub账号已注册并登录
  • ✅ 创建了公开仓库（Public）
  • ✅ 上传了所有问卷文件
  • ✅ 开启了GitHub Pages功能
  • ✅ 等待部署完成（有绿色标志）
  • ✅ 测试公网访问正常
